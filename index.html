Git Flow Branching Model

Este projeto segue o modelo Git Flow para organização de branches:

  • main: Branch principal com código de produção.
  • develop: Branch de desenvolvimento, integra features.
  • feature/*: Branches para novas funcionalidades (ex: feature/git-flow).
  • release/*: Branches para preparação de releases.
  • hotfix/*: Branches para correções urgentes em produção.

Atualmente estamos na branch hotfix/v3.0 corrigindo um problema crítico.
